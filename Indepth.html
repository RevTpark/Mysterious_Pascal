Into the Mysteries of Pascal Triangle
Lets learn about each property in depth.
Note: All the rows and elements are 0 indexed.

Sum of each Row

To create a pascal's triangle we add the elements as we go down the triangle, So instead we try to add them row-wise. While adding each row, we notice the pattern of the sums appears to be in powers of 2.
Example: 0th row sums to 1 => 20, 1st row sums to 2 => 21, 3rd row sums to 4 => 22, so on. And we obtain the sequence 1, 2, 4, 8, 16, 32, 64...

Unitray Sum of each row

When we add each number individually we obtain 2n for the given nth row. While one might think joining the numbers in the nth row also gives us 11n but it appears to be the case for the first 4 rows after that numbers don't seem to match 11n. The correct way is when we add the numbers in a unitary decimal fashion, we start to see the powers of 11, which also holds after the 4th row.
Example: 2nd row gives 100 + 20 + 1 = 121 => 112, 5th gives 100000 + 50000 + 100000 + 1000 + 50 + 1 = 1,61,051 => 115.

Line Symmetry

As the name suggests, this property concludes that Pascal Triangle is symmetric over the Y-axis or we can say it has mirror symmetry, ie, When viewed in a mirror the pascal's triangle will remain unaffected and appears the same. This property is more visual but has mathematical proof as well. In simple terms, the proof glances over the fact that nCr = nCn-r.

Combinatorics

Assume we have been given n objects and we need to pick r (where r <= n) objects from them. As we know, the number of ways to do so is nCr. We can use pascal's triangle to check this instantaneously by going to the nth row and checking the rth element. The vice versa also holds true, wherein it is impossible to draw an infinite Pascal's triangle, so nCr can be used to find the value of any element in the triangle.
Example: To find 5C2 we can go to the 5th row and check the 2nd element and we get 10, so we can conclude 5C2 = 10.

Binomial Expansion

One of the most used properties of pascal's triangle is obtaining Coefficients of Binomial Expansion. Assume we have to find the expansion of (a+b)n. We can go to the nth row and check elements.
Each element starting from left to right represents the coefficient of the expansion. Extra information required is expansion formula:

Example: To find (a+b)4, we can to check 4th row and obtain coefficients as 1, 4, 6, 4, 1. Now, we can construct the final answer as (a+b)4 = 1*a4 + 4*a3*b + 6*a2*b2 + 4*a*b3 + 1*b4.

Check Primes

Even though it might not look like there could be a relation between the prime numbers in the triangle but Prime Number verification can be done using Pascal's Triangle. Assume we have a number n and we want to verify if it is Prime or not. We can go to the nth row of Pascal's triangle and check if n is divisible by all the non-one elements(elements with value not equal to 1). If it is the number is Prime Number, otherwise, it is not a Prime Number. The vice versa also holds true, i.e., when the 1st element of a row is prime then all other elements are divisible by that number.
Example: n = 5, Now 5th row has elements 5,10,10,5. Since all elements are divisible by 5 we can verify that 5 is Prime.

Diagonals Magic

The diagonals of the Pascal Triangle also exhibit some weird properties.
The 1st diagonal contains elements of the same value equal to 1.
The 2nd diagonal contains all the Natural Numbers or Counting Numbers.
The 3rd diagonal contains all the Triangular Numbers. Triangular Numbers are a count of those numbers which can be arranged in an equilateral triangle.
The 4th diagonal contains Tetrahedral Numbers.
And so on...

Fibonacci Sequence

Even the Fibonacci Sequence makes an appearance in Pascal's Triangle. There seems no obvious pattern that leads to the Fibonacci Sequence, so how do we obtain it? If we consider the 45-degree diagonals and draw lines on the triangle, we start to notice the 1n diagonal starting from the top that contains a triangle element is [1], the 2nd diagonal contains [1], 3rd contains [1,1], 4th contains [1,2], 5th contains [1,3,1] and so on. We still don't see the Fibonacci Sequence just yet. Now we add all elements in each diagonal, We obtain 1, 1, 1+1 = 2, 1+2 = 3, 1+3+1 = 5
=> 1, 1, 2, 3, 5, 8 ... is the required Fibonacci Sequence.

Sierpinski’s Triangle

When we color all the odd terms in Pascal's Triangle, we obtain a fractal which is commonly known as Sierpinski’s Triangle. In Sierpinski’s Triangle, We start with an equilateral triangle and connect the midpoints of the segments to construct another triangle inside. This gives us access to 4 new triangles inside of the main triangle. On those triangles, we can repeatedly perform the earlier action to get new triangles within the previous triangles. This property is more easily understood visually.

Binary Form

We can make the triangle into binary form by getting the modulo of each element with 2. We will be left with 1's and 0's in the triangle. Now going row by row we can evaluate each binary number. Observing the first 9 rows as an example we get 1, 3, 5, 15, 17, 51, 85, 255, and 257. Here starting from 3 we can notice a pattern: 3 (Prime), 5 (Prime), 15 (5*3), 17 (Prime), 51 (17*3), 85 (17*5), 255 (17*15), 257 (Prime). Every number is Prime or multiplication of previous numbers. All the Prime numbers obtained are known as Fermat Primes, that is, they are of the type: 22n + 1.

Hexagonal Product

Consider any element in Pascal's Triangle that has 6 neighbors. According to the property, the product of any 3 alternate elements is equal to the remaining 3 alternate numbers. For better understanding, this property should be visualized.
Example: Consider the 4th row and its 1st element "4", the pairs of 3 alternate numbers are [3, 10, 1] and [6, 5, 1] and the product of each equals 30.

Hockey Stick

According to this property if we start from any of the outer 1's and follow the diagonal elements going down and at some point decide to stop then the element that is bottom right or left depending on where one started is the summation of all the elements in the path we followed. The path followed is the handle of the hockey stick and the element that holds the summation is the front part of the stick sort of in shape like L.
Example: Consider we start from 2nd row 2nd element and follow a path of 3 steps so we obtain the stick as [1, 3, 6]. Since we started to go down left the element to the bottom right contains 10 which also is the summation of all the path elements.

Obtaining π

Looking at Pascal's Triangle it seems to obtain Pi is a difficult task but when we take the multiplicative inverse of each number, starting from the top right the 2nd diagonal when elements are alternatively added and subtracted we obtain the answer as π - 2. Mathematically, 1 - (1/3) + (1/6) - (1/10) ... = π - 2. Source contains more in-depth into the mathematical proof of the same.

Euler's Number

We have tried summation of the rows and unitary summation to get different properties but we still haven't used the product of the rows to get some weird properties. Well, Euler's number is obtained by using the product of the rows. Consider the nth row and the product to be pn of that row, the product of (n-1)th row to be pn-1 , and (n+1)th row to be pn+1. Using this information we conclude that ((pn+1) * (pn-1)) / (pn2) = (1 + (1/n))n. Here as n tends to infinity, the value tends to Euler's Number e = 2.71.

Circle Mad max

Consider a circle with n points on its circumference, we connect all the points to each other. Now going to the nth row of Pascal's triangle, ignoring the first element, all the remaining elements represents the number of points, number of lines, number of quadrilaterals, number of pentagons, and so on in respective order.
Example: consider n = 5, now the 5th row elements starting from 1st row are 5, 10, 10, 5 and 1. This translates to circle 5 = number of points, 10 = number of lines, 10 = number of triangles,
5 = number of quadrilaterals and 1 pentagon.
